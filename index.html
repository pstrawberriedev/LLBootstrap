What is Bootstrap?!

    Open-Source
  • 100% Free - wow!
  • MIT-Licensed
  • Developed by really nice, swell people
  • Contributed to by scary strangers
    A Framework
  • Built upon LESS (ported to SASS)
  • Utilizing Javascript (jQuery)
  • Mainly used for its Grid System
  • Includes common JS Components
    A Quick-Start
  • Thoroughly tested & updated often
  • Excellent cross-browser support
  • Fully-customizable & extensible
  • Wonderful community support

Would this lady use Bootstrap?

    She wants...
  • A standard design
  • A responsive blog site
  • A slider on her home page
  • Different (grid) layouts for various pages
  • The ability to changes theme colors

Bootstrap can be used to create modern web layouts.

For this client, Bootstrap would supply everything she neeeded. Provided her design was built with a grid in mind, standard designs aren't difficult to flop into Bootstrap (with a bit of practice), whether it be from a .PSD/.AI/.IID; or an existing Site Reference.

Many WordPress blogs are built upon Bootstrap. This is because Bootstrap has become extremely popular among prototyping options, and can then be utilized to create an end result, should the occasion suit.

Bootstrap comes equiped with a plethora of modern web components. Among these are a slider (Bootstrap Carousel), grid presets (Containers, Rows, and Columns), as well as an easily-extensible CSS/LESS/SASS class system (as well as much more!)

What about Picard?

    He needs...
  • His Bridge Control UI updated
  • His ship's hull status overlaid on top of a 3D Vector
  • On-the-fly styling changes corresponding to his ship's Alert Level

Captain Jean-Luc Picard probably wouldn't choose Bootstrap.

The point is: Bootstrap isn't magic fearie dust. Though this has little to do with Bootstrap itself, we must always keep in mind that frameworks are about Control, Efficiency, and Consistency.

Could Picard implement Bootstrap for his Bridge Control UI? Of course! It would sparkle in the deep, sultry darkness of space. It would dazzle his alien visitors and bring them that much closer to signing a peace treaty. However, would it be practical to break his update process out to include a separate framework? Control, Efficiency, and Consistency may be at steak. (In modern context) What do you think?

It all comes down to this...

What is your skill set? Do you have time to learn a new framework?

    Beginners
  • Plentiful resources & community support
  • Pre-made & pre-coded themes
  • Javascript controlled mostly through HTML tags
  • Probably already being used in your WordPress Theme
    Designers & Light Front-End
  • Column positioning can be confusing - pushes & pulls
  • Paddings & margins often need tweaks
  • Takes a couple weeks to learn
  • Necessary to design for grid-style layouts
    Experienced Front-End
  • A bit heavy out of the box
  • May only want a certain part of Bootstrap (Carousel or the Grid)
  • May help your clients' sites to stay somewhat consistent

Which Path Will You Choose?

So far, this is what I've learned about Bootstrap in the last 2 years...

    The Setup is Quick
  • Customize what you want via web interface or via SASS/LESS
  • Include the CSS
  • Include the JS
  • Start laying out containers, rows, and columns
    After 1 Year
  • Include entire latest CSS + JS
  • Use defaults for everything, override what I need to
  • Use Carousel, Tabs, Accordion
  • Use Nav Menu & Nav Dropdowns
    After 2 Years
  • Ditch everything but the Grid and Carousel
  • Customize the Carousel as needed
  • Include basic overrides (especially for paddings & margins)
